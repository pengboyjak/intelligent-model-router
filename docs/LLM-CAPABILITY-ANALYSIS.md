## 二、模型详细优缺点

### 🔵 Claude Opus 4.8 (Anthropic) — 当前综合最强

```
AI 品质分: 61.4 ████████████████████████████ (#1 全球)
SWE-bench Pro: 69.2% ████████████████████ (超 GPT-5.5 10.6 分)
SWE-bench Verified: 88.6% ██████████████████████████
Terminal-Bench 2.1: 74.6% ██████████████████
OSWorld (电脑操作): 83.4% █████████████████████ (#1)
知识工作 Elo: 1890 █████████████████████████ (超 GPT 121 分, 胜率 67%)
幻觉率: ~36% (GPT-5.5 为 86%)
```

**优势:**
- **编码断层领先**: SWE-bench Pro 超 GPT-5.5 10.6 分。GPT 修 100 个 bug，Opus 修 118 个
- **代理编码稳定性**: 可生成数百并行子代理进行大规模重构
- **幻觉率最低**: 36% vs GPT 86%。模型被训练为表达不确定性而非编造
- **电脑操作最强**: OSWorld 83.4%。能操作 GUI、浏览器、文件管理器
- **对齐最佳**: 误对齐指数 1.9，前沿模型中最安全
- **生态广泛**: 驱动 GitHub Copilot、Cursor、Windsurf
- **1M 上下文统一价**: 无长上下文附加费 (Gemini 200K 以上翻倍)

**劣势:**
- **最贵**: $5/$25 per 1M tok。比 DeepSeek 贵 29 倍输出
- **终端落后 GPT**: Terminal-Bench 74.6% vs 82.7%
- **输出冗长**: 比 GPT 多用 30% 轮次完成相同任务
- **速度慢**: ~一半于 Gemini (~60 tok/s)
- **Tokenizer 膨胀**: Opus 4.7+ tokenizer 对相同文本多产 35% token
- **注意力衰减**: 50K token 后对早期约束的注意力下降

**最适合:** 多文件重构、代码审查、安全审计、法律文档、架构设计、企业代理

```
SWE-bench Pro: 58.6% ██████████████
OSWorld: 78.7% ███████████████████
幻觉率: ~86%
```

**优势:**
- **终端/CLI 最强**: Terminal-Bench 82.7%。Shell、Docker、DevOps 无敌
- **数学满分**: AIME 2025 100%。证明和推理能力顶尖
- **自主代理**: 长周期多步自主任务无需人工检查点
- **输出精简**: 比 Opus 省 token。适合 token 敏感的流水线
- **生态最深**: 最多插件、IDE 集成、ChatGPT 生态
- **翻译第一**: 10 语言往返翻译 8.69/10

**劣势:**

- **模型切换**: GPT-5.5 和 o4 之间的切换导致不一致

```

SWE-bench Pro: 55.4% ██████████████

```

**优势:**
- **极低成本**: 输出 $0.87/M。Flash 版 $0.28/M。比 Opus 便宜 29 倍
- **最大输出窗口**: 384K token 输出上限 (Opus 128K)
- **MIT 开源**: 完全可私有化部署
- **编码性价比最优**: 80.6% SWE-bench Verified
- **双 API 格式**: 原生支持 OpenAI 和 Anthropic API
- **推理效率**: 大规模上下文推理仅需 V3 的 27% 算力
- **国产芯片**: 华为昇腾 910C 训练

**劣势:**
- **数据隐私风险**: API 经中国服务器，受中国法律管辖
- **前沿差距**: 最难任务落后 Opus 15-20 分 SWE-bench

- **文档/创意弱**: 非编码任务能力有限
- **量化问题**: 多数 serverless 提供商使用 fp8 量化，输出质量下降
- **文学味淡**: "理工风格"，人文创作文采不足

**最适合:** 预算编码、数学推理、私有化部署、成本敏感批处理

## 六、2026 关键趋势与警示

1. **Fable 5 设定新标杆后暂停** — 最强模型因出口管制暂停，Opus 4.8 代位
2. **无单一赢家** — 编码(Opus)、终端(GPT)、多模态(Gemini)、推理(Grok)、成本(DeepSeek) 各有王者
3. **成本鸿沟 48 倍** — 相同负载 DeepSeek Flash $252/月 vs GPT-5.5 $12,000/月
4. **80% 集群拥挤** — 5 个模型 SWE-bench 差距 0.5 分内，价格差 20 倍
5. **基准≠现实** — GLM-5.2 基准高但代理崩溃。要在自己的工作流上测试
6. **脚手架比模型更重要** — 同一模型在不同评估套件下差距 17-21 分
7. **分层路由是 2026 标准模式** — 80% 调用走便宜模型, 20% 保留旗舰, 成本降 60-80%
8. **便宜反而更贵** — 有开发者月花 $1,800 用便宜模型反复重试; 一条旗舰 prompt 更划算
9. **版本锁定** — 新版≠更好。Gemini 曾版本更新导致用户文件被删
10. **上下文≠有效上下文** — 所有模型随输入增长而退化。关键信息放 prompt 开头
